최근에 프로젝트를 진행하면서 vanilla javascript + jquery 조합에서 벗어나여 프론트 엔드 프레임워크를 도입해봤다. 화면에서 비동기로 일어나는 이벤트가 너무 많아 일일이 DOM 렌더링을 관리할 자신이 없어서 도입하게 되었는데, 결과가 꽤 만족스러워서 다음 프로젝트부터는 아예 백엔드로 사용하고 있는 Laravel 과 통합하여 개발환경을 구성하면 어떨까 싶다.

많고 많은 프론트 엔드 프레임워크 중에서도 내가 택한건 VueJS 이 프레임워크인데, 일단 러닝 커브가 굉장히 적다. 공식 문서만 한번 쭉 훑으면 SPA 까지는 아니더라도, jquery처럼 필요한 부분에만 적용하여 사용하기 시작할 수 있는 수준이다. 소스가 가벼워서 부담이 없고, 나처럼 Laravel 을 사용하는 개발자라면 간단한 코드 몇줄이면 통합 환경이 구성된다는 장점이 있다.

  1. Laravel 설치

라라벨 한국어 매뉴얼 - 라라벨 코리아 커뮤니티(Laravel Korea Community)

오늘 (2018–04–20) 기준으로 최신 버전은 v.5.6 버전이다. 이미 한글 문서화가 다 되어 있기 때문에 따로 다루지 않겠다.

2. VSCode 설치

Download Visual Studio Code - Mac, Linux, Windows

개인적으로는 Atom 인터페이스가 마음에 들어 예전부터 사용해왔는데, VSCode를 사용해보고 난 후로는 다시 사용할 일이 없을 것 같다. 속도, 기능, 플러그인 수 까지 빠지는 것 하나 없는 혜자 IDE이다. 특히나 내장 NodeJS Debugger가 내장되어 있어 풀스택을 개발하는 사람들에게 유용하다.

3. VSCode 플러그인 설치

VueJS 파일 형식은 VSCode에서 기본으로 지원하는 파일형식이 아니기 때문에, 이를 위한 플러그인을 추가해주어야한다. 좌측 세로 탭에서 마지막 탭을 클릭한 후(마켓 플레이스), Vue를 검색하여 처음 나오는 플로그인을 설치 후 VSCode를 다시 로드하면 된다.

4. 프로젝트 설정

1번 링크에서 컴포저를 설치 하였다면, 프로젝트가 새로 만들어지기 원하는 폴더로 가서 아래와 같은 명령어를 입력한다.

composer create-project –prefer-dist laravel/laravel Boilerplate

뒤에 Boilerplate는 프로젝트 이름이니 바꿔주어야 한다. 위 명령어는 실행되는 데 몇 분정도 걸린다. 컴포저는 사용할 때마다 느끼는 거지만 너무 느리고 메모리도 많이 먹는다. npm 대신에 yarn 이 있듯이 composer도 경쟁 패키지 매니저가 나오면 하는 바람이 있다ㅜ

명령어가 수행되었다면, VSCode를 열어서 해당 폴더를 열어준다. 그러고 resources/assets/js/components 파일을 열어보면 짜잔~ 이미 ExampleComponent라는 vue 파일이 존재하고 있을 것이다. 사실 Laravel 에서는 VueJS 를 이미 내장하여 배포하고 있다. 우리가 해줄 일은 이 파일을 컴파일하고, view쪽에 집에 넣어 사용하기만 하면 된다. 일단 이 ExampleComponent 파일을 삭제하고, 같은 components 폴더 아래 HelloWorld.vue라는 파일을 하나 만든다. 내용은 아래와 같다.

이제 단축키인 Control + ` (윈도우 기준) 을 VSCode에서 누르면 프로젝트 디렉토리로 이동된 터미널이 생성된다. 터미널이 필요할 때마다 유용하게 사용할 수 있는 단축기이니 기억해두자. 이 터미널에 아래 명령어를 입력하여 Laravel 서버를 띄운다.

php artisan serve

이제 브라우저를 띄어서 localhost:8000으로 가보면

위와 같이 횡한 화면을 보게 될 것이다. 우리가 아직 새로 만든 HelloWorld 컴포넌트에 대해서 컴파일을 하지 않았기 때문에 렌더링이 되지 않은 것이다. 컴파일을 하기 위해서, Laravel 서버는 그대로 구동시켜놓고 터미널 패널 우측 + 아이콘을 클릭하여 새 터미널을 띄운 후 아래와 같은 명령어를 입력한다.

npm install

우선 컴파일 전에 javascript 용 필요한 패키지를 설치하는 작업이다. 1~2분 정도 걸린다. 위 작업이 완료되면 아래와 같은 명령어로 실제 컴포넌트 파일 컴파일을 수행할 수 있다.

npm run dev

작업이 완료되면, 컴파일 된 파일들이 출력된다. 다시 localhost:8000으로 돌아가서 페이지를 리로드 해보자.

짜잔~ 성공적으로 컴포넌트가 렌더링 되는 모습을 볼 수 있다. 기본 구성은 되었지만 더 간편화된 개발 환경 구성을 위하여 아직 밟아야 할 단계가 몇가지 더 있다.

HelloWorld.vue 파일로 돌아가 Hello Laravel + Vue! 문구를 No Thanks! 와 같은 다른 문구로 교체해본후에 localhost:8000을 로드해보면 화면은 여전히 Hello Laravel + Vue! 문구를 표시하고 있을 것이다. Vue파일은 수정될때마다 컴파일 되어야 변경사항이 반영된다. 그러나 우리가 일일이 파일을 수정하고 명령어를 날릴 수는 없는 노릇이기에, 이미 파일 변경을 자동으로 감지하여 컴파일을 다시 하는 명령어가 준비되어 있다.

npm run watch

위 명령어는 지속적으로 파일을 감시하여 컴파일을 다시 해준다. 명령어가 실행된 상태에서, 문구를 다시 바꾼 후 저장하고 localhost:8000을 리로드해보자. 변경 사항이 바로 반영된 것을 확인할 수 있을 것이다.

사실 여기까지만 구성해도 개발하는 데 별 무리는 없다. 나도 프로젝트를 이 구성으로 진행하였었다. 다만 현재 구성으로는 에러가 발생했을 시, 모든 에러가 app.js 파일을 가르키므로 오류 추적이 굉장히 어려워진다. VSCode의 디버깅 기능을 사용하여 어떻게 개발을 더 효율적으로 할 수 있는지 다음 포스트에서 더 살펴보겠다.
