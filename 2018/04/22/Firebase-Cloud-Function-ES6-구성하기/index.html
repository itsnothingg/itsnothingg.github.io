주의: 2018–04–15 기준으로 Cloud Function은 아직 Beta 버전이고 us-central 리전에서만 호스팅된다.

itsnothingg/firebase-cloud-es6-boilderplate

Firebase Cloud Function + ES6 환경이 끝나면 구성될 boilerplate 코드이다.

  1. Firebase Cloud Function 설정

시작하기: 첫 번째 함수 작성 및 배포 | Firebase

설정을 마쳤다면 알아차렸겠지만, Firebase Cloud Function은 Node v6.11.5에서 구동되기 때문에 ES6에 생긴 멋진 기능들을 사용할 수 없다. 이것은 엄청난 콜백 지옥을 경험할 수도 있다는 의미이기도 하다. 그러나 다행이도 우리에겐 Babel이라는 구원자가 있다. Babel은 js파일에 있는 ES6 문법을 ES5로 자동으로 구현하여 컴파일 해주는 아주 멋진 라이브러리다.

Babel · The compiler for writing next generation JavaScript

2. 프로젝트 구조 재구성

Cloud Function 과 Babel을 같이 사용하기 위해선 다음과 같은 프로젝트 구조를 가져야 한다.

.
├── src
│ ├── package.json 
│ ├── node_modules 
│ └── index.js 
├── dist
│ ├── package.json 
│ ├── node_modules 
│ └── index.js 
├── .firebaserc
├── firebase.json ├── node_modules
├── package.json
├── .babelrc

src 폴더에 원본 소스가 들어가게 된다. 1번에 있는 링크를 따랐다면 functions 폴더가 이 폴더로 바뀌어야한다.

dist 폴더는 src에서 babel을 통해 컴파일 된, 실제로 Cloud Function에 배포될 소스가 생성된다. 이 폴더는 자동 생성될 것이므로 신경쓰지 않아도 된다.

우리가 원하는 것을 구현하기위해 수정되거나 생성되어야 하는 파일들은 루트에 있는 package.json, .bablerc, firebase.json 정도 이다.

3. 의존성 추가

npm이 아니라 yarn을 사용하여 의존성을 관리하였다.

루트 폴더로 이동한 후 다음과 같은 명령어를 날려준다.

yarn init

몇가지를 물어볼 것인데 이 구성은 중요하지 않으니 엔터로 넘어간다.

후에 다음과 같은 명령어로 babel 의존성을 추가한다.

yarn add @babel/preset-env @babel/cli @babel/core -D

또한 자동 빌드를 도와줄 두가지 라이브러리를 추가로 설치한다.

yarn add copyfiles rimraf-D

4. 파일 수정

2번에서 보았던 루트에 있는 .babelrc를 추가할 차례다. 내용은 아래와 같다.

{
 "presets": [
 ["@babel/env", {
 "targets": {
 "node": "6.11.5"
 }
 }]
 ]
}

.firebase.json에서 배포될 폴더를 새로 지정하여 준다.

{
 "functions": {
 "source": "dist"
 }
}

package.json에 몇가지 스크립트를 추가해준다.

"scripts": {
 "deploy": "yarn run build && firebase deploy –only functions",
 "prebuild": "rimraf dist && copyfiles -u 1 -a \"src\\" -e node_modules dist",,
 "build": "babel src –out-dir dist –ignore src/node_modules",,
 "postbuild": "cd dist && yarn",
 "watch": "babel src –out-dir dist –ignore src/node_modules –watch"
}

deploy 는 배포할 때 실행될 스크립트다. 내부적으로 build 명령어를 실행하고 firebase의 배포를 실행한다.

prebuild는 dist폴더를 우선 삭제하고, node_modules를 제외한 src에 있는 파일들을 모두 복사한다.

build에서 실제로 babel이 컴파일을 수행하여 src에 있는 모든 js 파일들을 dist에 덮어쓴다.

postbuild에서는 dist 폴더에 의존성을 설치한다.

watch 명령어는 실행시켜 놓으면 실시간으로 src에 있는 js 파일에 변경을 감지하여 컴파일 후 dist에 저장한다.

5. 배포

이제 다음과 같은 명령어가 원본 소스를 ES6로 재컴파일 하여 Firebase에 자동으로 배포하는 모습을 볼 수 있을 것이다.

yarn run deploy
